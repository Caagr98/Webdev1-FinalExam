C98

The Invention of the Web

The Beginning

In the beginning, there was nothing.

Well, since it wasn't the absolute beginning, there was a world and all that, but nothing that matters.

Getting bored with this nothing, man created computers. And man saw that it was good.

Man was wrong, of course; those computers were nothing remarkable with today's standards.

However, at the time, today's standards were still the future, and with that today's standards, it was good.

That's beside the point though.

The computer was a neat invention, but then man decided to create the Internet, too.

The W3

After the beginning, it was fairly common to write scientific articles on computers, usually with various (nowadays primitive) markup languages with support for things like headings and italics.

CERN, for example, used a language called SGMLguid.

This language supported most things needed at the moment, but there was no way to link between different articles except simply saying the name and telling the reader to find it himself.

Therefore, in 1990, a CERN worker named Tim Berners-Lee (TimBL) invented the World Wide Web (originally abbreviated W3, now WWW), along with HTML and HTTP to implement it; the major feature of HyperText compared to other markup languages was the ability to HyperLink to other articles, creating a Web of documents.

The first recorded HTML document[1] is clearly identifiable as HTML, but the structure is completely different - instead of a tree, it appears to use a linear structure. Other early HTML documents[2][3] also display this behavior, along with some non-standard tag names, most of which come from the previously mentioned SGMLguid.

The list of supported tags for this ancient version of HTML is archived here. Interestingly, TimBL expressed dissatisfaction at the <H{1..6}> tags, instead wanting a nested <SECTION> tag containing a generic <H> tag - one of which was added about 20 years later, while the other still doesn't exist.

Adoption

A few important points about the Web are that it's free and decentralized. Being free meant that anyone could use it for anything they want, and being decentralized means that there isn't one single server hosting the entire Web, which would have meant that the maintainer of that server would have total control over every website.

In January '93, there were about fifty web servers worldwide. By October the same year, there were over five hundred.

Early browsers include the original unnamed browser for the NeXT operating system, some (nowadays rather unknown) browsers called Erwise and ViolaWWW, and the console-based Lynx (which still exists today, but it doesn't have very many users).

However, the browser that made the Web what it is today is Mosaic. Featuring an intuitive UI, a Windows version, and support for inline images (<img> tags), it quickly got popular.

The Browser Wars

Browsers

The Mosaic browser was licensed to multiple companies, who created their own forks, including AirMosaic and Spyglass Mosaic. The creator of the original Mosaic founded a company, Mosaic Communications Corporation, and created Mosaic Netscape.

To avoid legal trouble, Mosaic Communications Corporation was renamed to Netscape Communications Corporation (NCC), and the browser was renamed to Netscape Navigator, usually known as Netscape.

Being both fast, free (for non-commercial use), and easy to use, Netscape Navigator dominated the World Wide Web by 1995.

In 1995, Internet Explorer 1.0 was released, reaching feature parity with version 3.0 in 1996.

Of course, it also had a bunch of features not found in Netscape, including the (nowadays much hated) <blink> and <marqee> tags.

In 1996, IE version 4 was released. Due to being preinstalled on all Windows computers - a vast majority - IE's market share quickly grew.

While both NN and IE used HTML, they had some differences (for example the aforementioned blink and marqee tags). This led to some websites to add logos saying "This website works best in Netscape" or similar, which meant that to have the optimal browsing experience, you'd have to switch between the two browsers depending on what website you were visiting - which is hardly optimal.

To resolve this, it was agreed that websites should try to follow the standards set by the World Wide Web Consortium (W3C).

War

Microsoft had two major advantages over NCC:

  • NCC had no major products other than Netscape. Microsoft had, among other things, Windows. This meant Microsoft had far superior income, and could put more money into their browser, and provide it for free to all users, not only non-commercial users.
  • IE was bundled with every copy of Windows, which had a 90% market share. Most users were also rather new to the internet, and didn't even know that there were any other browsers.

In 2002, IE had 96% market share: far more than Netscape ever had.

Thus ended the First Browser War.

After being defeated by IE, Netscape open-sourced their browser, and later entrusted it to the Mozilla Foundation.

In 2003, Microsoft discontinued IE for Mac. A few months later, with the release of OSX 10.3, Apple released the Safari browser, along with the WebKit rendering engine, which was later ported to Windows in 2007.

After a few years, and about as many name changes (from Phoenix to Firebird to Firefox), Mozilla Firefox 1.0 was released in 2004, bringing numerous new features (many of them, for example a separate seach bar, copied from Opera).

In 2006, Microsoft released IE7, which contained tabs, a separate search bar, and better standards support - all of which had existed in Firefox and Opera for a long time.

Google released Google Chrome, also based on WebKit, in 2008. After releasing Mac and Linux versions, its market share grew rapidly.

In mid-2010, IE dropped below 50% market share for the first time since its creation, mainly due to the fast growth of Chrome.

Some sources claim that by mid-2012, Chrome had surpassed IE in popularity, though other sources disagree.
